## вњ¦ First-time Setup

```bash
# 1. Install dependencies
npm install

# 2. Add platforms
cordova platform add android
cordova platform add ios        # macOS only

# 3. Install plugins (declared in config.xml вЂ” Cordova reads them automatically)
cordova plugin add cordova-plugin-geolocation
cordova plugin add cordova-plugin-network-information
cordova plugin add cordova-plugin-splashscreen
cordova plugin add cordova-plugin-statusbar
cordova plugin add cordova-plugin-inappbrowser
cordova plugin add cordova-plugin-vibration
# cordova plugin add cordova-plugin-wkwebview-engine   # macOS only
cordova plugin add cordova-plugin-file

# 4. Set your Google Maps API key in src/index.html
#    Replace YOUR_API_KEY on this line:
#    src="https://maps.googleapis.com/maps/api/js?key=YOUR_API_KEY&..."
#
#    Required APIs to enable in Google Cloud Console:
#      вњ… Maps JavaScript API
#      вњ… Places API
#      вњ… Directions API
#      вњ… Geocoding API  (for "use my location" reverse geocode)
#
#    Restrict the key to:
#      Android apps: your package name  za.co.siphika.chauffeur
#      iOS apps:     your bundle ID     za.co.siphika.chauffeur

# 5. Build Vite в†’ www/
npm run build

# 6. Prepare Cordova (copies www/ into platform projects)
cordova prepare
```

## вњ¦ Production Builds

### Android APK / AAB

```bash
# 1. Generate a signing keystore (once)
keytool -genkey -v \
  -keystore siphika-release.keystore \
  -alias siphika \
  -keyalg RSA -keysize 2048 \
  -validity 10000

# 2. Build release APK
npm run android:rel

# The signed APK will be at:
# platforms/android/app/build/outputs/apk/release/app-release.apk

# 3. For Google Play, build AAB:

./gradlew bundleRelease
```

## вњ¦ Cordova Plugins & What They Do

| Plugin                               | Used for                                    |
| ------------------------------------ | ------------------------------------------- |
| `cordova-plugin-geolocation`         | "Use my location" button on booking map     |
| `cordova-plugin-network-information` | Offline banner when no internet             |
| `cordova-plugin-statusbar`           | Dark status bar, gold-aware background      |
| `cordova-plugin-splashscreen`        | Hides native splash after app boots         |

| `cordova-plugin-file`                | Receipt download (future feature)           |

## вњ¦ Key Cordova Adaptations Made

| Topic           | What changed                                                               |
| --------------- | -------------------------------------------------------------------------- |
| **Boot**        | `DOMContentLoaded` в†’ `deviceready` event                                   |
| **Script type** | Bundled as ES module; `window.*` exports make inline onclick handlers work |
| **Asset paths** | All `./` relative вЂ” works under `file:///android_asset/`                   |
| **Vite output** | `base: './'`, `format: 'iife'`, outputs to `www/`                          |
| **CSP**         | Full policy in both `<meta>` and `config.xml`                              |
| **Viewport**    | `user-scalable=no`, `viewport-fit=cover` for notches                       |
| **Safe areas**  | `env(safe-area-inset-*)` CSS vars + JS fallback                            |
| **Back button** | Android hardware back wired to `goBack()` / exit                           |
| **Network**     | Offline banner via `cordova-plugin-network-information`                    |
| **Haptics**     | Vibration pattern on booking confirm                                       |
| **Geolocation** | "Use my location" в†’ reverse geocode в†’ fills pickup input                   |
| **StatusBar**   | `StyleBlackTranslucent` + `#0A0A0C` background                             |
| **Touch**       | `-webkit-tap-highlight-color: transparent`, active press states            |
| **Scroll**      | `-webkit-overflow-scrolling: touch`, `overscroll-behavior-y: none`         |

## вњ¦ Google Maps API Key Setup

1. Go to [console.cloud.google.com](https://console.cloud.google.com)
2. Create a new project or select existing
3. Enable: **Maps JavaScript API**, **Places API**, **Directions API**, **Geocoding API**
4. Create credentials в†’ API Key
5. Restrict to Android/iOS app by package name
6. Replace `YOUR_API_KEY` in `src/index.html`
